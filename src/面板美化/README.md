# 修仙TRPG面板美化脚本库

根据 `prompt.txt` 中的规定，本库包含了所有 **主要面板类型** 的美化脚本。

## 🎯 快速开始

### ⭐ 推荐方案：使用通用 action_info 模板

如果你需要美化 `action_info` 面板，**强烈推荐使用通用模板**：

- **文件**: `action_info美化.html`
- **特点**:
  - ✅ 一个文件处理所有 action_info 类型
  - ✅ 自动识别面板类型并应用对应配色
  - ✅ 使用正则表达式提取：`/<action_info>((?:(?!<\/?action_info>)[\s\S])*?)<\/action_info>/gs`
  - ✅ 支持批量处理多个面板
  - ✅ 易于集成和扩展

**详细文档**: 📖 [action_info通用模板说明.md](action_info通用模板说明.md)

## 📋 完整面板类型总览

### 1. **item_info** - 物品/功法/法宝/傀儡/灵兽信息
- **文件**: `item_info美化.html`
- **用途**: 展示物品、功法、法宝、傀儡、灵兽的详细信息
- **主要标签**: 品质、类型、属性值、标签、效果
- **配色**: 金色主题 (#d4af37)

### 2. **char_info** - 角色信息
- **文件**: `char_info美化.html`
- **用途**: 展示NPC或其他角色的完整信息
- **主要标签**: 境界、种族、灵根、体质、技艺
- **配色**: 紫色主题 (#e9d8fd)

### 3. **secret_realm_info** - 秘境信息
- **文件**: `secret_realm_info美化.html`
- **用途**: 展示秘境的难度、类型、危害、奖励等信息
- **主要标签**: 难度等级、类型、危害、奖励
- **配色**: 橙色主题 (#f6ad55)

### 4. **action_info** - 行动面板（通用模板）⭐
- **文件**: `action_info美化.html` (推荐)
- **用途**: 展示所有类型的行动面板
- **支持的子类型**:
  - ⚔️ 战斗行动 (battle)
  - 🔨 生产制作 (craft)
  - 📖 修为获取 (cultivation)
  - ⚡ 突破晋升 (breakthrough)
  - 🧠 参悟领悟 (insight)
  - ⚖️ 非战斗判定 (judge)
- **配色**: 根据类型动态应用
- **详细说明**: 📖 [action_info通用模板说明.md](action_info通用模板说明.md)

### 4.1 **action_info (战斗)** - 战斗行动（独立版本）
- **文件**: `action_info_battle美化.html`
- **用途**: 展示战斗中的各个阶段（战况总览、行动顺序、攻击行动、伤害计算等）
- **主要标签**: 攻方、守方、行动、伤害、命中、闪避
- **配色**: 红色主题 (#fc8181)
- **备注**: 如果使用通用模板，不需要此文件

### 4.2 **action_info (生产)** - 生产制作（独立版本）
- **文件**: `action_info_craft美化.html`
- **用途**: 展示炼丹、炼器、制符等生产制作过程
- **主要标签**: 生产准备、制作检定、生产结算、品质、投入物、产出
- **配色**: 绿色主题 (#68d391)
- **备注**: 如果使用通用模板，不需要此文件

### 4.3 **action_info (修为获取)** - 修为获取（独立版本）
- **文件**: `action_info_cultivation美化.html`
- **用途**: 展示修炼、服丹、双修、夺灵等修为获取方式
- **主要标签**: 修为方法、修为进度、天谴值、境界
- **配色**: 蓝色主题 (#90cdf4)
- **备注**: 如果使用通用模板，不需要此文件

### 4.4 **action_info (突破)** - 突破晋升（独立版本）
- **文件**: `action_info_breakthrough美化.html`
- **用途**: 展示突破晋升过程（突破判定、历劫、突破结算）
- **主要标签**: 结果、历劫类型、目标境界、难度
- **配色**: 橙色主题 (#f6ad55)
- **备注**: 如果使用通用模板，不需要此文件

### 4.5 **action_info (参悟)** - 参悟领悟（独立版本）
- **文件**: `action_info_insight美化.html`
- **用途**: 展示参悟秘籍、阵图、石碑等的过程
- **主要标签**: 参悟检定、参悟结果、大成奖励、进度
- **配色**: 紫色主题 (#b794f4)
- **备注**: 如果使用通用模板，不需要此文件

### 4.6 **action_info (非战斗判定)** - 非战斗判定（独立版本）

- **用途**: 展示非战斗状态下的各类判定（神念感知、体魄强压、机巧解谜、威压交涉）
- **主要标签**: 检定类型、核心资质、结果

## 🚀 使用方法

### 方案 1: 使用通用 action_info 模板（推荐）⭐

```javascript
// 在你的代码中
const text = `
  <action_info>
  {战况总览}
  | 人物: 李道玄 | 境界: 筑基中期 |
  </action_info>

  <action_info>
  {攻击行动}
  | 攻方: 李道玄 | 伤害: 250 |
  </action_info>
`;

// 调用美化函数
processActionInfo(text);
```

**优点**:
- ✅ 一个文件处理所有 action_info 类型
- ✅ 自动识别面板类型
- ✅ 支持批量处理
- ✅ 易于集成

**详细文档**: 📖 [action_info通用模板说明.md](action_info通用模板说明.md)

### 方案 2: 使用独立面板美化脚本

1. 在 HTML 文件中，将 `$0` 占位符替换为实际的面板内容
2. 面板内容应遵循以下格式：
```html
<item_info>
{物品名称}
| 品质: 天 | 类型: 法宝 | 攻击力: 100 |
| 效果: 火焰伤害+50% |
</item_info>
```

3. 集成到项目：
   - 复制对应的 HTML 文件到项目中
   - 在需要美化的地方引入该 HTML
   - 通过 iframe 或直接嵌入的方式使用

### 面板格式规范
- 使用 `{标题}` 表示面板标题（独占一行）
- 使用 `| 内容 |` 表示数据行（以 `|` 开头和结尾）
- 支持的标签格式：`标签名: 值`

## 🎯 最佳实践

1. **保持一致性**: 同类型的面板使用相同的配色
2. **清晰的层级**: 使用不同大小和颜色区分重要程度
3. **响应式设计**: 确保在各种屏幕尺寸上都能正常显示
4. **可访问性**: 使用足够的对比度确保可读性
5. **性能优化**: 使用 CSS 过渡而不是 JavaScript 动画
